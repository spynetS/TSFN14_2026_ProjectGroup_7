#+title: Readme

On the backend we use a MVC pattern.

- we have the models in the *src/models* folder
- views in the *src/routes*
- controllers in the *src/controllers*

Each model should have its own view (/route/) and controller.


** To deploy on minikube

#+begin_src bash
minikube start
eval $(minikube docker-env)
docker build -t fittnessduel-backend:latest ./
minikube image load fittnessduel-backend:latest
kubectl apply -f ./pod.yml
kubectl apply -f ./service.yml
#+end_src
